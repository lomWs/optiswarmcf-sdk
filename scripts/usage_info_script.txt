# SCRIPTS USAGE GUIDE

## Overview

The `scripts/` directory contains utility scripts to manage the project workflow, including:

* Python SDK setup
* ROS2 backend build
* Experiment execution
* Backend lifecycle management
* Runtime logs

The project is organized into two main components:

* `sdk/`
  Python SDK package (`optiswarmcf`)
* `backend_ros2/`
  ROS2 workspace containing runtime backend nodes

The scripts are designed to simplify setup and reduce environment/configuration errors.

---

# 1. setup_sdk.sh

## Purpose

Initialize the Python SDK environment.

## Usage

```bash
./scripts/setup_sdk.sh
```

## What it does

* Creates a Python virtual environment in:

```text
sdk/.venv
```

(if it does not already exist)

* Activates the virtual environment
* Upgrades `pip`
* Installs the SDK in editable mode:

```bash
pip install -e .
```

## When to use

* First setup after cloning the repository
* After deleting or recreating the virtual environment
* After modifying SDK dependencies

## Notes

* This installs only the Python SDK
* This does NOT build ROS2 packages
* This does NOT start the backend
* ROS2 environments must still be sourced when running experiments

---

# 2. build_backend.sh

## Purpose

Build the ROS2 backend workspace.

## Usage

```bash
./scripts/build_backend.sh [optional_workspace_path]
```

## What it does

* Sources the ROS2 environment
* Resolves ROS dependencies using `rosdep`
* Builds the workspace using `colcon`

## When to use

* After cloning the repository
* After modifying backend ROS2 code
* After adding ROS dependencies
* If ROS packages are missing

## Notes

* This does NOT start backend nodes
* Workspace must contain a valid ROS2 `src/` directory
* Generated install artifacts are placed inside:

```text
backend_ros2/install/
```

---

# 3. run_experiment.sh

## Purpose

Run a complete experiment using the ROS2 backend and a Python controller.

## Usage

```bash
./scripts/run_experiment.sh <algorithm.py> [optional args]
```

## Example

```bash
./scripts/run_experiment.sh examples/minimal_controller.py
```

## What it does

* Verifies backend and SDK availability
* Sources:

  * ROS2
  * backend workspace
  * SDK virtual environment
* Starts backend components if not already running
* Runs the controller script inside the SDK environment

## Backend components

Depending on configuration, the backend may start:

* VRPN client
* mocap_bridge_ros2
* cf_bridge

## Logs

Logs are stored in:

```text
.runtime/logs/
```

Typical log files:

* `vrpn.log`
* `mocap.log`
* `cf_bridge.log`

## Notes

* Backend processes run in background
* Backend restart may occur if configuration changes are detected
* Backend health checks are minimal and configuration-dependent
* Use `stop_backend.sh` to terminate backend processes

---

# 4. stop_backend.sh

## Purpose

Stop all backend processes started by the runtime scripts.

## Usage

```bash
./scripts/stop_backend.sh
```

## What it does

* Stops backend processes
* Cleans internal PID tracking files

## When to use

* To manually terminate the backend
* Before changing backend configuration
* Before rebuilding backend components

## Notes

* Does NOT delete logs
* Does NOT remove runtime state

---

# 5. show_logs.sh

## Purpose

Monitor backend logs in real time.

## Usage

```bash
./scripts/show_logs.sh
```

## What it does

Streams backend logs using:

```bash
tail -f
```

## Typical logs

* `vrpn.log`
* `mocap.log`
* `cf_bridge.log`

## When to use

* Debugging
* Runtime monitoring
* Backend diagnostics

## Stop monitoring

```text
Ctrl + C
```

---

# Typical Workflow

## 1. Setup SDK (once)

```bash
./scripts/setup_sdk.sh
```

The SDK virtual environment will be created in:

```text
sdk/.venv
```

---

## 2. Build backend

```bash
./scripts/build_backend.sh
```

This generates:

```text
backend_ros2/install/
```

---

## 3. Run an experiment

```bash
./scripts/run_experiment.sh examples/minimal_controller.py
```

The script automatically prepares:

* ROS2 environment
* Backend workspace
* Python virtual environment

---

## 4. Monitor logs (optional)

```bash
./scripts/show_logs.sh
```

---

## 5. Stop backend

```bash
./scripts/stop_backend.sh
```

---

# SDK Structure

The SDK uses a modern Python package layout:

```text
sdk/
├── pyproject.toml
├── src/
│   └── optiswarmcf/
└── .venv/
```

The SDK is installed in editable mode during development.

---

# Important Notes

* ROS2 must already be installed on the system
* Backend must be successfully built before running experiments
* The SDK virtual environment must exist
* Runtime logs are stored in:

```text
.runtime/logs/
```

* ROS dependencies such as `rclpy` are managed through ROS2 (`package.xml`, `rosdep`, apt packages), not through `pip`

---

# Recommended Environment Order

When manually activating environments:

```bash
source /opt/ros/<distro>/setup.bash
source backend_ros2/install/setup.bash
source sdk/.venv/bin/activate
```

Example:

```bash
source /opt/ros/jazzy/setup.bash
source backend_ros2/install/setup.bash
source sdk/.venv/bin/activate
```
