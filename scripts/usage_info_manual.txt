# SDK SETUP AND RUN GUIDE

## FIRST TIME SETUP (only once)

### 1. Go to the repository root

```bash id="c1a8i8"
cd ~/Scrivania/testSotware
```

---

### 2. Setup the SDK environment

```bash id="l1i3rq"
./scripts/setup_sdk.sh
```

This script automatically:

* Creates the virtual environment in:

```text id="v0v49l"
sdk/.venv
```

* Activates the environment
* Upgrades `pip`
* Installs the SDK in editable mode

---

### 3. Build the ROS2 backend

```bash id="0m4lrf"
./scripts/build_backend.sh
```

This generates the ROS2 install workspace:

```text id="44s2q3"
backend_ros2/install/
```

---

# DAILY WORKFLOW

## 1. Open a new terminal

---

## 2. Run an experiment

```bash id="kxbv3n"
./scripts/run_experiment.sh examples/minimal_controller.py
```

The script automatically:

* Sources ROS2
* Sources the backend workspace
* Activates the SDK virtual environment
* Starts backend components if needed
* Runs the controller

---

# MANUAL RUN (advanced/debugging)

If you want to manually run controllers without `run_experiment.sh`:

## 1. Source ROS2

```bash id="5mhbci"
source /opt/ros/$ROS_DISTRO/setup.bash
```

---

## 2. Source backend workspace

```bash id="e0lddj"
source ~/Scrivania/backendProvas/backend_ros2/install/setup.bash
```

---

## 3. Activate SDK environment

```bash id="ns2j8t"
cd ~/Scrivania/backendProvas/sdk
source .venv/bin/activate
```

---

## 4. Run controller

```bash id="1yjx4w"
python examples/minimal_controller.py
```

---

# NOTES

* ROS2 must already be installed
* Backend must be built before running experiments
* The SDK virtual environment must exist
* ROS dependencies such as `rclpy` are managed by ROS2, not by `pip`
* Runtime logs are stored in:

```text id="8a34f2"
.runtime/logs/
```

---

# RESET SDK ENVIRONMENT (if needed)

If the Python environment becomes corrupted:

```bash id="y6khm6"
cd ~/Scrivania/backendProvas/sdk
rm -rf .venv
```

Then rerun:

```bash id="zgh1ja"
cd ~/Scrivania/backendProvas
./scripts/setup_sdk.sh
```
